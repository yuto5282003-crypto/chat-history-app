# SLOTY — リスク一覧とガード

## 1. 安全リスク

### R-S1: 対面時の身体的安全

| 項目 | 内容 |
|------|------|
| リスク | 対面スロットで犯罪・ハラスメントが発生する |
| 影響度 | 致命的 |
| 発生確率 | 中 |
| ガード | |
| G1 | 初期は公共の場所のみ（カフェ・駅・公園） |
| G2 | 詳細位置は予約成立後にのみ開示 |
| G3 | 深夜帯（22:00-6:00）は本人確認済みのみ |
| G4 | 予約画面に「緊急通報」ボタン（110番導線） |
| G5 | 通報→即ブロック→運営レビュー |
| G6 | 未成年（18歳未満）は対面利用不可 |
| 将来 | チェックイン/チェックアウト機能、同行モード |

### R-S2: ストーキング・つきまとい

| 項目 | 内容 |
|------|------|
| リスク | 特定ユーザーのスロット出品パターンから行動を推測される |
| 影響度 | 高 |
| 発生確率 | 中 |
| ガード | |
| G1 | カレンダー・生活パターンが推測可能な情報を出さない（原則P3） |
| G2 | 同一ユーザーが同一出品者のスロットを連続購入するとフラグ |
| G3 | ブロック機能（即時遮断。ブロックした相手のスロットは非表示） |
| G4 | 出品者のスロット一覧は表示しない（個別スロットカードのみ） |
| 将来 | 行動パターン異常検知 |

### R-S3: 恋愛/性的サービスの隠れ提供

| 項目 | 内容 |
|------|------|
| リスク | スロットやSquare投稿を使って恋愛/性的サービスを提供する |
| 影響度 | 高 |
| 発生確率 | 高 |
| ガード | |
| G1 | 禁止事項の明文化（利用規約） |
| G2 | テキストフィルタ（NGワード検知） |
| G3 | 通報→自動非表示（閾値3件）→運営レビュー |
| G4 | カテゴリを限定（恋愛系カテゴリは提供しない） |
| 将来 | AI テキスト分類 |

| 影響度 | 高 |

| ガード | |

## 4. 技術リスク

### R-T1: 衝突チェックの不整合

| 項目 | 内容 |
|------|------|
| リスク | 同時リクエストで同一時間帯に複数予約が成立する |
| 影響度 | 高 |
| 発生確率 | 中 |
| ガード | |
| G1 | DBトランザクション（SERIALIZABLE or SELECT FOR UPDATE） |
| G2 | 楽観的ロック（version カラム） |
| G3 | Slot ステータス更新は原子操作で実行 |
| テスト | 同時予約の競合テスト（並列リクエスト） |

### R-T2: チケット残高の不整合

| 項目 | 内容 |
|------|------|
| リスク | チケットの二重消費や不正増加 |
| 影響度 | 高 |
| 発生確率 | 低 |
| ガード | |
| G1 | 全操作を ticket_ledger に記録（台帳ベース） |
| G2 | 冪等性キー（idempotency_key）で重複防止 |
| G3 | トランザクション内で残高チェック→消費を実行 |
| G4 | 日次バッチで ledger合算 vs キャッシュ残高を検証 |
| テスト | 並列チケット消費テスト |

### R-T3: 決済トラブル

| 項目 | 内容 |
|------|------|
| リスク | 決済失敗・二重決済・返金失敗 |
| 影響度 | 高 |
| 発生確率 | 低 |
| ガード | |
| G1 | Stripe Webhook で決済状態を正確に追跡 |
| G2 | 冪等性キーを全決済リクエストに付与 |
| G3 | エスクロー方式（完了後に入金） |
| G4 | 返金失敗時のリトライ + アラート |
| テスト | Stripe テストモードでの全フロー検証 |

### R-T4: パフォーマンス（スロット検索）

| 項目 | 内容 |
|------|------|
| リスク | 位置ベース検索が遅くなる |
| 影響度 | 中 |
| 発生確率 | 低（初期は仙台のみで小規模） |
| ガード | |
| G1 | PostgreSQL の空間インデックス（PostGIS 将来） |
| G2 | 初期はジオハッシュベースのフィルタで十分 |
| G3 | ページネーション（デフォルト20件） |
| G4 | 検索結果キャッシュ（30秒TTL） |
