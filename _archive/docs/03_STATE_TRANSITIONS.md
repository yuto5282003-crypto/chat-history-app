# SLOTY — 状態遷移図

---

## 1. Slot（スロット）状態遷移

```
                    ┌──────────┐
                    │  DRAFT   │
                    │ （下書き） │
                    └────┬─────┘
                         │ [出品ボタン + 衝突チェックOK]
                         ▼
                    ┌──────────┐
               ┌───>│  LISTED  │<──────────────────┐
               │    │（公開中） │                    │
               │    └──┬──┬──┬─┘                    │
               │       │  │  │                      │
               │       │  │  │ [期限切れ]            │
               │       │  │  └──────────────┐       │
               │       │  │                 ▼       │
               │       │  │           ┌──────────┐  │
               │       │  │           │ EXPIRED  │  │
               │       │  │           │（期限切れ）│  │
               │       │  │           └──────────┘  │
               │       │  │                         │
               │       │  │ [売り手が手動停止]        │
               │       │  │ [非公開予定と衝突検知]    │
               │       │  ▼                         │
               │       │ ┌──────────┐               │
               │       │ │  PAUSED  │───────────────┘
               │       │ │（販売停止）│ [再開ボタン]
               │       │ └──────────┘
               │       │
               │       │ [買い手が購入]
               │       │
               │       ├─[即確定の場合]───────────────┐
               │       │                             ▼
               │       │                      ┌──────────────┐
               │       │                      │   BOOKED_    │
               │       │                      │  CONFIRMED   │
               │       │                      │  （成立）     │
               │       │                      └──────┬───────┘
               │       │                             │
               │       └─[承認制の場合]──┐            │
               │                        ▼            │
               │                 ┌──────────────┐    │
               │                 │   BOOKED_    │    │
               │                 │   PENDING    │    │
               │                 │（承認待ち）    │    │
               │                 └──┬──┬──┬─────┘    │
               │                    │  │  │          │
               │ [タイムアウト]      │  │  │          │
               ├────────────────────┘  │  │          │
               │                       │  │          │
               │  [売り手が拒否]        │  │          │
               ├───────────────────────┘  │          │
               │                          │          │
               │  [売り手が承認]           │          │
               │                          └──────────┘
               │
               │                             │
               │                             │ [実施完了]
               │                             ▼
               │                      ┌──────────────┐
               │                      │  COMPLETED   │
               │                      │  （完了）     │
               │                      └──────────────┘
               │
               │                             │ [キャンセル]
               │                             ▼
               │                      ┌──────────────┐
               │                      │  CANCELLED   │
               │                      │（キャンセル） │
               │                      └──────────────┘
               │
               └── ※BOOKED_PENDING→LISTED: 拒否/タイムアウト時、
                   スロットを再公開（売り手が希望する場合）
```

### 遷移条件まとめ

| From | To | トリガー | 条件 |
|------|----|---------|------|
| DRAFT | LISTED | 出品ボタン | 衝突チェックOK |
| LISTED | PAUSED | 手動停止 or 衝突検知 | — |
| LISTED | EXPIRED | 期限切れ | start_at < now |
| LISTED | BOOKED_PENDING | 買い手が購入（承認制） | 与信成功 |
| LISTED | BOOKED_CONFIRMED | 買い手が購入（即確定） | 決済成功 |
| PAUSED | LISTED | 再開ボタン | 衝突なし |
| BOOKED_PENDING | BOOKED_CONFIRMED | 売り手が承認 | SLA内 |
| BOOKED_PENDING | LISTED | 売り手が拒否 or タイムアウト | 再公開希望時 |
| BOOKED_PENDING | CANCELLED | タイムアウト | SLA超過 |
| BOOKED_CONFIRMED | COMPLETED | 実施完了 | end_at < now + 両者確認 |
| BOOKED_CONFIRMED | CANCELLED | キャンセル | キャンセル規定適用 |

## 2. Booking（予約）状態遷移

```
┌──────────┐
│   INIT   │
│（初期化） │
└────┬─────┘
     │ [決済処理開始]
     ▼
┌──────────────────┐
│    PAYMENT_      │
│   AUTHORIZED     │
│  （与信完了）     │
└────┬─────────────┘
     │
     ├─[即確定スロットの場合]──────────────┐
     │                                    ▼
     │                             ┌──────────────┐
     │                             │  CONFIRMED   │
     │                             │  （確定）     │
     │                             └──────┬───────┘
     │                                    │
     └─[承認制スロットの場合]──┐           │
                               │           │
     ┌─────────────────────────┘           │
     ▼                                    │
(Slot=BOOKED_PENDING)                     │
     │                                    │
     ├─[売り手が承認]─────────────────────┘
     │
     ├─[売り手が拒否]──────┐
     │                     ▼
     │              ┌──────────────────┐
     │              │  CANCELLED_BY_   │
     │              │     SELLER       │
     │              │（売り手キャンセル）│
     │              └────┬─────────────┘
     │                   │
     │                   ▼
     │              ┌──────────┐
     │              │ REFUNDED │
     │              │（返金済み）│
     │              └──────────┘
     │
     ├─[タイムアウト]──────┐
     │                     ▼
     │              ┌──────────────────┐
     │              │  AUTO_CANCELLED  │
     │              │（自動キャンセル）  │
     │              └────┬─────────────┘
     │                   │
     │                   ▼
     │              ┌──────────┐
     │              │ REFUNDED │
     │              └──────────┘
     │
     │  CONFIRMED から:
     │
     ├─[開始時刻到来]──────────┐
     │                         ▼
     │                  ┌──────────────┐
     │                  │ IN_PROGRESS  │
     │                  │ （進行中）    │
     │                  └──────┬───────┘
     │                         │
     │                         │ [終了時刻 + 完了確認]
     │                         ▼
     │                  ┌──────────────┐
     │                  │  COMPLETED   │
     │                  │  （完了）     │
     │                  └──────────────┘
     │                    ↓ [売り手へ入金処理]
     │
     ├─[買い手がキャンセル]────┐
     │                         ▼
     │                  ┌──────────────────┐
     │                  │ CANCELLED_BY_    │
     │                  │    BUYER         │
     │                  │（買い手キャンセル）│
     │                  └────┬─────────────┘
     │                       │
     │                       ▼
     │  ┌──────────────────────────────────┐
     │  │ キャンセル時刻に基づき:            │
     │  │ 24h以上前 → REFUNDED（全額）      │
     │  │ 12-24h   → REFUNDED（全額）      │
     │  │ 6-12h    → PARTIAL_REFUNDED(50%) │
     │  │ 6h未満   → 返金なし              │
     │  └──────────────────────────────────┘
     │
     └─[売り手がキャンセル(確定後)]──┐
                                     ▼
                              ┌──────────────────┐
                              │ CANCELLED_BY_    │
                              │    SELLER        │
                              └────┬─────────────┘
                                   │
                                   ▼
                              ┌──────────┐
                              │ REFUNDED │ (全額 + ペナルティ)
                              └──────────┘
```

### 遷移条件まとめ

| From | To | トリガー |
|------|----|---------|
| INIT | PAYMENT_AUTHORIZED | Stripe PaymentIntent 与信成功 |
| PAYMENT_AUTHORIZED | CONFIRMED | 即確定 or 売り手承認 |
| PAYMENT_AUTHORIZED | CANCELLED_BY_SELLER | 売り手拒否 |
| PAYMENT_AUTHORIZED | AUTO_CANCELLED | SLAタイムアウト |
| CONFIRMED | IN_PROGRESS | start_at 到来 |
| CONFIRMED | CANCELLED_BY_BUYER | 買い手キャンセル |
| CONFIRMED | CANCELLED_BY_SELLER | 売り手キャンセル |
| IN_PROGRESS | COMPLETED | end_at + 完了確認 |
| CANCELLED_* | REFUNDED | 返金処理完了 |
| CANCELLED_* | PARTIAL_REFUNDED | 部分返金処理完了 |

## 3. Request（広場依頼）状態遷移

```
┌──────────┐
│   SENT   │
│ （送信済）│
└──┬──┬──┬─┘
   │  │  │
   │  │  │ [24h経過]
   │  │  └──────────────┐
   │  │                 ▼
   │  │          ┌──────────┐
   │  │          │ TIMEOUT  │
   │  │          │（タイムアウト）│
   │  │          └────┬─────┘
   │  │               │
   │  │               ▼
   │  │          ┌──────────┐
   │  │          │ REFUNDED │ (全額: 5🎫)
   │  │          └──────────┘
   │  │
   │  │ [受信者が拒否]
   │  └──────────────┐
   │                 ▼
   │          ┌──────────┐
   │          │ REJECTED │
   │          │（拒否）   │
   │          └────┬─────┘
   │               │
   │               ▼
   │          ┌──────────────────┐
   │          │ PARTIAL_REFUNDED │ (半額: 2🎫)
   │          └──────────────────┘
   │
   │ [受信者が承認]
   ▼
┌──────────┐
│ ACCEPTED │
│（承認）   │
└────┬─────┘
     │
     ▼
[Slot自動生成] → [Booking作成] → Booking状態遷移へ
```

### 遷移条件まとめ

| From | To | トリガー | チケット処理 |
|------|----|---------|------------|
| SENT | ACCEPTED | 受信者が承認 | 消費確定（返金なし） |
| SENT | REJECTED | 受信者が拒否 | 半額返金（2🎫） |
| SENT | TIMEOUT | 24h未応答 | 全額返金（5🎫） |

## 4. Friend（フレンド）状態遷移

```
┌──────────────┐

       │ [QR/リンク]
       ▼

│   PENDING    │
│ （承認待ち）  │
└──┬──┬────────┘
   │  │
   │  │ [トークン期限切れ / 未応答]
   │  └──────────────┐
   │                 ▼
   │          ┌──────────────┐
   │          │  (消滅/無効)  │
   │          └──────────────┘
   │
   │ [双方が承認]
   ▼
┌──────────────┐
│   ACCEPTED   │◄─────────────────┐
│ （フレンド）  │                   │
└──┬───────────┘                   │
   │                               │
   │ [ブロック]                     │ [ブロック解除]
   ▼                               │
┌──────────────┐                   │
│   BLOCKED    │───────────────────┘
│ （ブロック中）│
└──────────────┘
```
