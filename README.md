# SLOTY — 時間共有マーケット

時間をスロット単位で売買できるマーケットプレイスアプリ。
「すれ違い」で今ヒマな人と即マッチ。チケット制でシンプルな課金体験。

## デモモード（UI確認用）

DB・認証・Stripe の設定不要でUI/動線を確認できます。

### ローカル起動

```bash
cp .env.example .env.local
npm install
npm run dev
```

ブラウザで http://localhost:3000 を開く（スマホ幅で確認推奨）。

### Vercel デプロイ手順

1. GitHub にリポジトリを push（済み）
2. [vercel.com](https://vercel.com) にログイン → **Add New Project**
3. GitHub リポジトリを選択してインポート
4. **Environment Variables** に以下を追加：
   - `NEXT_PUBLIC_DEMO_MODE` = `1`
   - `OWNER_EMAIL` = `demo@sloty.app`
   - `ADMIN_PASSCODE` = `sloty2024`
   - `RETENTION_DAYS` = `180`
5. **Deploy** をクリック
6. デプロイ完了後、発行された URL をスマホで開いて確認

## Ownerが /admin に入るための設定

| 環境変数 | 説明 | デモ値 |
|---------|------|--------|
| `OWNER_EMAIL` | Owner判定用メールアドレス | `demo@sloty.app` |
| `ADMIN_PASSCODE` | 管理画面ログインパスコード | `sloty2024` |
| `RETENTION_DAYS` | KYC画像保管期限（日） | `180` |

管理画面ログイン手順（URL直打ちのみ、ユーザーUIからの導線なし）：
1. ブラウザで `https://<your-domain>/admin` を直接開く
2. パスコード入力（デモ: `sloty2024`）
3. 30分間のセッションが開始
4. 各タブ（本人確認 / ユーザー / 通報 / セッション / 会話ログ / チケット / 監査ログ）で管理
5. ※ ユーザー向けUI（設定画面等）に管理画面への導線は表示されません

## テスト公開前 確認手順（4項目）

### 1. すれ違いマップUI（/nearby）
- `/nearby` を開く → 地図がフル画面に近い表示で表示される
- 上部に半透明ステータスバー（「近くのチェックイン」「◯人がヒマ」）
- 中央に自分のアバター（丸型＋グロー）
- 下部に3つの丸ボタン（🌏 POI / 📍 現在地 / 💬 受信箱）
- 🌏ボタンでPOI表示ON/OFF → カテゴリフィルタ（カフェ/ご飯/公共/公園）
- 写真未登録時：赤帯ではなくカード型案内UI
- チェックイン → 近くの人リストからピン送信（5🎫）

### 2. 課金確認（/profile）
- `/profile` → 「チケットを購入」ボタン → パック選択（10/30/80/200🎫）
- **DEMO_MODE=1**: 購入ボタンで即チケット加算 → 台帳に記録される
- **本番（DEMO_MODE=0）**: Stripe Checkoutに遷移する旨のアラート表示
  - 実際のStripe連携は `STRIPE_SECRET_KEY` 設定後に `/api/checkout` を実装
- サブスク（Basic/Plus）: 加入で即チケット付与（デモ）/ Stripe Billing（本番）

### 3. 本人確認（/verify）
- `/verify` → Lv1（セルフィー）/ Lv2（身分証＋ライブネス）を選択
- **カメラ撮影**: 「📷 カメラで撮影」→ 端末カメラ起動
- **写真フォルダ**: 「📁 写真から選択」→ ファイルピッカー
- 撮影ガイド枠（顔の楕円/カードの矩形）を薄く表示
- 画像は最大1024pxに圧縮 → sessionStorageに保存（メタはlocalStorage）
- 提出後2秒でAIスコア → `/admin` → 本人確認タブで管理者が承認

### 4. カレンダー自動反映（/bookings → /calendar）
- `/market` → スロット詳細 → 予約（即確定の場合）
  - → `/bookings` に「📅 カレンダー済」バッジ表示
  - → `/calendar` に「SLOTY：通話（30分）」等のbusy予定が自動追加
- ピン承認（/pings）/ 依頼承認（/requests/inbox）でも同様にカレンダー反映
- 仕様：
  - タイトル: `SLOTY：{通話|対面}（{N}分）`
  - 種別: busy / すれ違い対象外: ON / バッファ: 前後10分
  - eventId = `booking:{bookingId}` で二重登録防止
  - キャンセル時: カレンダー予定も削除

- **新規登録**: `/signup` → 確認メール送信 → `/dev/mailbox` でリンクをタップ → メール認証完了 → 自動ログイン
- **ログイン**: `/login` → emailVerified=true のみログイン可 → 未確認は再送ボタン表示
- **パスワードリセット**: `/forgot` → outbox → `/reset-password?token=...` で新PW設定
- **パスワード変更**: `/account` → リセットメール送信 → outbox → リンクで変更
- **メール変更**: `/account` → 新メール入力 → outbox → リンクで確定
- パスワードはハッシュ化保存（平文保存/表示禁止）
- セッション有効期限: 24時間
- **DEMO_MODE=1**: 認証ガードスキップ / **DEMO_MODE=0**: 未ログインは `/login` リダイレクト

- 目的タグ、趣味タグ、活動エリア、連絡スタイル、NG事項

  - Step2: プロフィール写真セット

## 設計方針

- **チケット制**: UIに現金(¥)表記は一切なし。全てチケット🎫で統一
- **時間が主役**: カレンダー→空き時間→マッチが基本フロー
- **eKYC必須**: Lv0=閲覧のみ、Lv1=通話OK、Lv2=対面OK
- **安全設計**: 予約紐付きルーム・時間窓制限・外部連絡先マスキング・通報/ブロック
- **プライバシー**: 位置情報はチェックイン時のみ取得、マーカーにジッター付与、距離は範囲表示
- **管理者最小権限**: KYC画像/チャット閲覧は理由必須・監査ログ・DL禁止・ウォーターマーク
- **パスワード保護**: 平文閲覧不可・強制リセットのみ
- **AI提案層**: `src/lib/ai.ts` にルールベース実装。将来LLM差し替え可能
- **デモモード**: `src/lib/demo-store.ts` でlocalStorage完結。本番DB差し替え前提
